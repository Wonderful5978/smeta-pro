# Смета-Про — сайт + PWA

Статический сайт. GitHub Pages, без сервера/БД/npm.

## Структура

```
site/
├── index.html                  ← лендинг (продажа, цена, СБП-инструкция)
├── demo/                       ← ПУБЛИЧНАЯ демо-версия с водяным знаком
│   ├── index.html              ← приложение с watermark
│   ├── manifest.json
│   ├── sw.js
│   └── icons/
├── r2026-fkx8m9q/              ← ПРИВАТНАЯ полная версия (URL знают только покупатели)
│   ├── index.html              ← полное приложение (БЕЗ watermark)
│   ├── install.html            ← страница «Спасибо за покупку + инструкции»
│   ├── manifest.json, sw.js, icons/
│   └── files/
│       └── smeta-pro.apk       ← Android (подписан постоянным keystore)
├── .gitignore
└── README.md
```

## Важно про приватную папку

Имя `r2026-fkx8m9q` неугадываемо, но если кто-то опубликует эту ссылку — продукт станет бесплатным. Чтобы при таком обнаружении быстро «закрыть» утечку:

1. **Переименовать папку** `r2026-fkx8m9q` → `r2027-xxxxxxxx` (новый случайный код)
2. **Заменить все ссылки** в письмах клиентам, на лендинге (если они есть), в скрипте отправки
3. **Удалить старую папку** через git push
4. **Перевыпустить APK** через PWABuilder с новым URL

Это не идеальная защита, но достаточно для дешёвого продукта (1990 ₽) с лояльной аудиторией.

## Деплой на GitHub Pages

Уже залито на https://github.com/Wonderful5978/smeta-pro и включено в Settings → Pages → main / root.

Обновления: меняешь файлы → `git push` → через 1–2 мин на сайте.

**Для PWA-обновлений:** при изменении файлов в `demo/` или `r2026-fkx8m9q/` меняй `CACHE_NAME` в соответствующем `sw.js` — иначе у уже установивших клиентов останется старая версия из кэша.

## APK для Android

Файл `r2026-fkx8m9q/files/smeta-pro.apk` подписан постоянным keystore (см. `_tools/keystore/`).

⚠️ **APK сгенерирован для URL `/app/`** — после переезда папки `app/` → `r2026-fkx8m9q/` старый APK будет показывать 404. Нужно пересобрать через PWABuilder с URL:

```
https://kpsmeta.ru/r2026-fkx8m9q/
```

Полная инструкция: `../_tools/HOW-TO-UPDATE-APK.md`.

## Локальная проверка

Простейший локальный сервер:
```bash
cd site
python -m http.server 8000
```
Открой `http://localhost:8000/`.
